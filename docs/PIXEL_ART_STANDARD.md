# 五行迷宮塔防 — 像素美術提升規格

> 參考：`Sample/` 中的粗輪廓角色、塔樓建造階段與投石塔多方向動畫。

## 決策

不再把先前淘汰的 AI 64px 圖作為正式預設。正式戰鬥使用程式矩陣像素精靈，外置圖片只保留實驗切換與載入 fallback。

世界格維持 16 單位，避免重寫 A*、地圖、建塔與存檔座標；美術不受限於一格，可用 24×24、24×32 或 32×32 畫布跨格繪製。

## 正式精靈標準

- 外輪廓：1 個來源像素的暖黑／深棕輪廓，任何背景下都能辨識。
- 色階：每種材質至少暗部、主色、亮部三階，不以 Glow 代替明暗。
- 塔樓：基礎塔需具有共同石台／木台輪廓，元素核心放在塔頂；Lv2 與合成塔改變建築剪影。
- 怪物：普通怪建議 16×16 至 24×24；Boss 建議 24×32 或 32×32。
- 動畫：待機至少 2 幀、攻擊或行走至少 2–4 幀；改變姿態而非只做整張縮放。
- 陰影：使用小型實色像素陰影；大量單位不使用雙層模糊 Glow。
- UI 圖示：由正式戰鬥矩陣精靈生成，避免另一套圖示風格。

## 尺寸與碰撞

- `TILE_SIZE` 是世界座標與尋路單位，不等於精靈畫布上限。
- 每種敵人可設定 `visualScale` 與 `collisionRadiusTiles`。
- Boss 龍影目前使用 `visualScale: 1.55`、`collisionRadiusTiles: 0.75`。
- 子彈進入敵人的圓形受擊半徑即命中；血條與克制提示圈依視覺比例同步放大。
