## 開發原則

1. **每次提交可遊玩**：不引入破壞主選單→戰鬥→結算流程的變更
2. **資料驅動優先**：新內容先寫 JSON 配置，再補判定邏輯，最後做 UI
3. **測試覆蓋**：每個新模組至少 15 項測試（正常路徑 + 邊界 + migration）
4. **單一職責**：純函式放在 state.ts，UI 邏輯放在 ui/，配置載入放在 config.ts
5. **視覺迭代**：所有 UI 變更需在 Antigravity 驗證畫面效果
