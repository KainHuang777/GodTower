# 美術 UI 改造與程式核心稽核

> 日期：2026-07-12  
> 狀態：第一輪實作完成，建置與自動測試通過

## 結論

目前 TypeScript＋Canvas 2D 核心足以支撐核准的明亮像素美術與 PC／Mobile 橫向 UI，不需要改用 WebGL，也不需要重寫戰鬥核心。

真正會限制品質與效能的不是 Canvas 技術本身，而是部分靜態內容仍在逐幀處理、同一精靈使用多次 Glow 重畫，以及舊 UI 樣式集中在大型 HTML 並大量使用行內覆寫。第一輪已先處理高收益、低風險項目。

## 第一輪已完成

### 視覺

- 建立獨立 `visualRefresh.css`，集中管理新一代視覺 Token 與覆寫規則。
- 全域改用跨裝置清楚黑體 fallback，不強制像素字。
- 主選單改為通天塔剪影、明亮山水遠景與大型原生遊戲按鈕。
- 戰鬥 HUD 改為暖色像素面板、大數字資訊徽章、大型塔快捷列與高權重「下一波」。
- 天賦頁改為明亮經脈畫面，加入太極核心、人體五行背景與大型分支頁籤。
- 中式標準戰場改為日間草地與暖色沙石路。
- 五行色由暗底霓虹改為明亮自然色。
- Mobile 第一階段限定橫向，保留 48px 以上主要觸控區與既有平移／縮放機制。

## Mockup 對齊補完（第二輪）

第一輪仍沿用舊 DOM 排版，造成實際畫面只有配色接近、結構仍像 Web Prototype。第二輪已補上：

- 主選單接入無文字的通天塔山水高密度像素背景；圖片缺失時仍有 CSS 山水 fallback。
- 戰鬥畫面改成單一全畫面 Canvas，生命／金幣／波次／擊殺與控制列皆覆蓋在戰場上。
- 七座塔改為底部大型圖像快捷列；常用的速度、圖鑑與下一波保持常駐。
- 召喚、路線、診斷與離開收進「更多」抽屜，移除上下兩排 Web 工具列的視覺主導性。
- 天賦頁加入中央經脈工作區與右側詳情面板；點選穴位後才由明確的「升級」按鈕確認。
- 五行分支依人體／太極核心重新分布，不再只是橫向卡片列表。
- 戰場預設改用 64px 高密度像素精靈；資產載入失敗時自動回退原生矩陣精靈。
- 27 個塔與怪物高密度精靈已加入 production public assets，避免正式建置缺檔後全部退回低像素版本。

### 效能

- 靜態地圖障礙物已合併進 Offscreen Canvas 地圖快取，不再每幀掃描整張網格並重畫。
- 怪物與砲塔的雙層 Glow 改為單層低模糊輪廓光，每個單位每幀少一次精靈重畫。
- 日間場景改以輪廓與色塊提供辨識，不再依賴高成本大範圍陰影。
- 保留既有事件驅動地圖快取更新；建塔、拆塔、尋路或切換主題時才重建。

## 核心風險評估

| 項目 | 目前判定 | 對品質／效能的影響 | 後續處理 |
|---|---|---|---|
| Canvas 2D | 可繼續使用 | 目前不是改造阻擋 | 不遷移 WebGL |
| 地圖 Tile | 已有整圖快取 | 正向；每幀只需一次地圖 `drawImage` | 維持事件驅動失效 |
| 靜態障礙物 | 已改善 | 已由逐幀繪製移入地圖快取 | 實機確認建拆塔後同步 |
| 單位 Glow | 已改善 | 大量單位時原本會造成重複繪製與陰影成本 | 維持單層，Boss／選取狀態才允許額外光效 |
| 粒子系統 | 中風險 | 大量命中、死亡與天氣同時出現可能過量 | 下一輪加入總量上限與畫質分級 |
| 路線動態箭頭 | 中低風險 | 每幀依完整路徑迴圈繪製 | 低階 Mobile 可降更新頻率或關閉流動動畫 |
| 天氣 | 中風險 | 雨、霧、雷電可能與粒子競爭預算 | 特殊關卡啟用，Mobile 預設降低密度 |
| CSS／DOM | 維護風險 | 大量行內樣式會阻礙後續一致性 | 新樣式集中外部檔案，逐畫面移除舊覆寫 |
| 外置資產 | 可控 | 缺檔可能影響視覺但不應中斷遊戲 | 維持圖片與音訊 fallback 規範 |

## 尚未執行、但建議的第二輪核心改善

1. 為命中、死亡、天氣粒子設定分級上限；PC 使用完整密度，Mobile 使用中等密度。
2. 將路線流光更新限制為約 20–30 FPS，戰鬥主循環仍維持 60 FPS。
3. 將只與塔類型相關的光環預渲染，避免每座塔每幀重畫多個橢圓。
4. 對長局進行 20 波記憶體快照，確認粒子、彈道、飄字 DOM 與音效池均能回收。
5. 逐步把 `index.html` 的行內樣式移到元件化 CSS，避免後續改色需要大量 `!important`。

這些項目不阻擋目前 UI 改造。應先完成標準示範關卡的畫面驗收，再根據實際 FPS 與裝置結果決定優先順序。

## 仍需人工畫面驗收

自動測試無法判斷實際視覺品質。下一步需在 PC 與至少一台橫向 Mobile／模擬尺寸檢查：

- 主選單通天塔是否被標題與按鈕遮擋過多。
- 標準日間關卡中，路徑、障礙物、塔與怪物剪影是否清楚。
- 1280×720 下 HUD 是否壓縮戰場過多。
- Mobile 橫向下七座塔與「下一波」是否能穩定點按。
- 天賦頁太極、人體輪廓與技能卡是否形成清楚的先後層級。
