# 天賦經脈頁：概念圖視覺對齊第二階段

> 狀態：P2-A～P2-E 已完成；使用者已推進至 P3 Gate A
> 更新日期：2026-07-16
> 範圍：只調整天賦頁的顯示名稱、敘事資料、HTML／CSS／SVG 與互動呈現；不改 `TalentId`、數值、前置條件、解鎖門檻或存檔格式。

## 1. 設計定位

本階段將目前「功能完整、視覺較抽象」的天賦頁，推進為可讀的古代經脈圖譜。建議世界觀定調為：

**《黃帝內經》藏象語彙 × 後世任督脈學 ×《周易》陰陽意象的東方幻想介面。**

三層素材必須清楚分開：

| 層級 | 可用內容 | 文案規則 |
|:---|:---|:---|
| 《黃帝內經》可直接支持 | 木肝、火心、土脾、金肺、水腎；五色、五志；五藏官職；營衛；「陰平陽秘」 | 可標明篇名；短句須核對原文，不把玩法效果寫成古籍結論。 |
| 後世經絡／脈學發展 | 任督作為奇經八脈、任督循行與相關修煉意象 | 使用「取意」「化用」；不宣稱全部是《內經》逐字原句。 |
| 《易》象與遊戲化創作 | 太極、兩儀、八卦、境階、五行塔戰鬥職能 | 標為易學意象或遊戲轉譯；不可統稱《黃帝內經》典故。 |

醫學與文化限制：

- 不把五藏敘事當成現代解剖、生理、診斷或養生建議。
- 不把雙魚太極圖、八卦、丹田或小周天稱為《黃帝內經》原圖或原術。
- 若顯示實際穴位名稱，位置必須大致正確；無把握時改用抽象節點名並標記 `需要確認`。
- 古書感標題可用「五藏」，新手提示補充「即五臟」。
- 水行原典色彩是黑／玄；介面可混入水藍提高辨識，但不可說原典以亮藍代表水。

## 2. 資訊架構與命名

### 頁面與四章

- 頁面正式名稱：**靈蘭藏象圖**（2026-07-14 已確認）。
- 新手副標：**五行經脈・天賦修習**。
- 四章：
  - 第一境・**任督築基**（原「基礎」，0 累積點）
  - 第二境・**氣機初行**（原「氣血」，2 累積點）
  - 第三境・**五藏應象**（原「五行」，6 累積點）
  - 第四境・**陰陽合化**（原「陰陽」，12 累積點）

境階屬遊戲化命名，不宣稱為古籍原有修煉分級。頁籤需同時保留白話副標，避免新手只看見古語。

### 命名原則

每個節點固定顯示兩層：

1. 古典主名：負責世界觀與辨識。

`TalentId`、效果、等級上限、成本與依賴全部保持不變；只新增顯示用 metadata。

### 五藏敘事卡

| 五行／五藏 | 原典官職取意 | 遊戲轉譯 | 視覺標記 |
|:---|:---|:---|:---|
| 木／肝 | 將軍之官，謀慮出焉 | 佈局、控場、持續傷害 | 青綠、葉、筋脈線 |
| 火／心 | 君主之官，神明出焉 | 爆發、節奏、主動強化 | 赤、火、脈搏光 |
| 土／脾 | 脾胃為倉廩之官，五味出焉 | 築防、資源、穩定 | 黃土、山、城垣 |
| 金／肺 | 相傅之官，治節出焉 | 精準、穿透、規律 | 白／銀金、刃、節律紋 |
| 水／腎 | 作強之官，伎巧出焉 | 減速、蓄勢、持久 | 玄黑＋水藍、波、骨紋 |

詳細卡應分成「藏象取意」與「戰鬥效果」兩區，避免 lore 與規則混在一起。脾的官職原文同時包含脾胃，若只顯示「脾」須明確寫「取意」。

等級可顯示「一重・初啟／二重・行氣／三重・通脈／四重・周流／五重・圓滿」，並保留 `等級 n/5` 數字。此階序同樣是遊戲化創作。

## 3. 視覺構圖

### 古書外框與頁面層級

- 外層：深木框或青銅框、回紋／雲雷紋角飾。
- 內層：象牙紙、墨線雙欄、淡紙纖維、少量朱砂批註與篆印。
- 標題：方形玉印＋「靈蘭藏象圖」；天賦點改成玉牌。
- 頁籤：像裝訂在書頁上的章籤；啟用頁籤要與紙面連在一起。
- 高飽和金色只用於可學習／已選節點，不鋪滿全頁。

### 中央經脈圖譜

- 正面與背面人體必須分離構圖；正式視覺採手繪古書風生成圖，responsive inline SVG 保留為資產載入失敗時的可用 fallback。
- 任脈走正面腹中線，採月白／青白；督脈走背部脊柱，採金紅／日光色。
- 五藏印記放在可辨識的人體區域，標示「肝木、心火、脾土、肺金、腎水」。
- 太極位於中央核心；用 SVG path 繪製，不依賴字型中的 `☯`。
- 八卦只作低對比外圈水印或鎖定環，不與五行主節點競爭。

### 節點與連線

- 五行主節點改為八角／菱形印章：金上、木左、火右、水左下、土右下，環繞太極。
- 元素色之外必須有圖形提示：葉、火、山、刃、波；不能只靠顏色辨識。
- 已解鎖路徑為實線；可學習為細金邊與緩慢呼吸；鎖定為淡墨虛線與封泥鎖印；滿級加朱印「成」。
- 外圈未開放節點若尚非真實 `TALENT_TREE` 內容，必須標為 inert 的「未開放章節」，不可偽裝成可點擊天賦。

### 右側詳情章頁

固定順序：元素印記 → 古典主名 → 機制副標 → 等級 → 藏象取意 → 戰鬥效果 → 前置／花費 → 升級按鈕。選取節點時，詳情框、人體部位與對應經線要同步高亮，但點擊「升級」前不得更動遊戲狀態。

## 5. 驗收條件

- 第一眼可辨識古書框架、人體經脈、任督二脈、五藏、太極與低調八卦，而非只有抽象底板。
- 五行頁在概念圖相同的大方向呈現五個印章節點、中央太極、五色路徑與外圈鎖定節點。
- 16 個現有天賦全部可進入；鎖定、可學習、已解鎖、滿級、選取五種狀態不只靠顏色區分。
- 詳情卡永遠顯示正確的主名、機制副標、等級、典故取意、實際效果、前置條件與按鈕狀態。
- 舊存檔、0／2／6／12 累積點門檻、重置後分支不倒退、教學自動選取與所有戰鬥效果不變。
- 外置圖片缺失時仍可完整使用天賦頁。
- 1680×944、1366×768、1024×768、896×414、844×390 無重疊或不可控的雙向捲動。
- 自動測試與 `npm run build` 通過；Antigravity 確認人體輪廓、經線對比、節點點擊範圍與桌面／橫向 Mobile 構圖。
