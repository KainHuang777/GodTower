## 1. 問題陳述

### 現狀
| 指標 | 當前值 | 目標（Roguelike 標準） |
|------|--------|----------------------|
| 首局可達波次 | Wave 20（通關） | Wave 3-5 |
| 全天賦滿級時間 | ~5 小時（19 局） | 60-70 小時 |
| 首次通關 Wave 20 | ~5 小時 | 40+ 小時 |
| Ascension 20 通關 | 不存在 | 150+ 小時 |
| 跨局成長類型 | 純數值（全部） | 數值 + 內容 + 機制 |

### 根因分析
1. **天賦點獲取過快**：`max(1, floor(wave/3))` → Wave 20 單局 6 點
2. **總需求過低**：~111 點即可全滿
3. **無分級難度**：僅一套難度，打通即畢業
4. **跨局獎勵單一**：全部是數值加成，無新機制/新配方解鎖

## 2. 核心設計哲學

### 三層成長軸（取代純數值）
```
永恆層（天賦）: 數值成長 + 配方解鎖 + 機制解鎖
隨機層（卡牌）: 每局 3 選 1，產生局間差異
挑戰層（Ascension）: 自願增加難度，解鎖炫耀獎勵
```

### 關鍵原則
1. **內容漏斗 > 數值漏斗**：70% 的 72 小時來自「策略空間擴展」，非「怪物 HP 提升」
2. **失敗 = 學習**：每局 Game Over 都產出至少一個「下次知道該做什麼」的 insight
3. **軟 Gate > 硬 Gate**：不阻止玩家進入高波次，但讓未準備好的玩家自然失敗，且有收穫

## 3. 天賦曲線重構

### 3.1 天賦點獲取新公式

```ts
/** 基於存活波次 + 突破個人紀錄 + 里程碑獎勵 */
function calcTalentPointsEarned(
  survivedWaves: number,
  personalBest: number,
  milestonesReached: number[]
): number {
  // 基數：wave/4（較當前 wave/3 慢 25%）
  let points = Math.max(1, Math.floor(survivedWaves / 4));

  // 突破個人紀錄獎勵：每超越 2 波多 1 點
  if (survivedWaves > personalBest) {
    points += Math.floor((survivedWaves - personalBest) / 2);
  }

  // 里程碑首次獎勵（各一次）
  for (const m of [5, 10, 15, 20]) {
    if (survivedWaves >= m && !milestonesReached.includes(m)) {
      points += 3;
    }
  }

  return points;
}
```

**效果對比**：
| 存活波次 | 當前 | 新公式（基數） | 新公式（含 PB+里程碑） |
|:-------:|:---:|:------------:|:------------------:|
| 3 | 1 | 1 | 1 |
| 10 | 3 | 2 | 5（含 PB + milestone） |
| 20 | 6 | 5 | 8（含 PB + milestone） |

### 3.4 雙軌天賦點制（刷關 vs 挑戰成就）

#### 核心哲學
玩家不能靠「反覆刷低難度關卡」累積天賦點來解鎖一切。必須在**安全但低收益**與**高風險但高回報**之間做選擇。

#### 雙軌機制

| 軌道 | 類型 | 天賦點獲取 | 機制 |
|:---:|:----|:---------:|:-----|
| **Track A** | 刷關（Ascension 0-5） | 僅基數公式 `floor(wave/4)+1` | 無成就獎勵，僅存活波次計點 |
| **Track B** | 挑戰（Ascension 6+ 或條件模式） | 基數公式 + **成就獎勵** | 首次完成成就可獲得一次性天賦點 |

#### 三類成就

| 類別 | 示例 | 獎勵點 | 觸發 |
|:----|:----|:-----:|:----|
| **波次成就** | 首次打通 Ascension 6/8/10/12/15/20 | 3-20 | 該 Ascension 首次通關 |
| **條件成就** | 單元素通關、無牆通關、零失血、速通 | 3-8 | 滿足條件自動觸發 |
| **累積成就** | 解鎖全配方、合成 100 次、擊殺 1000 怪 | 5-10 | 累積計數達到 |

3. **選擇壓力**：玩家刷 Ascension 3 安全得 2 點 vs 挑戰 Ascension 7 可能得 3+8=11 點

#### 預期行為曲線
```

第 30-60 局： 有意識地追特定成就（無牆流、單元素流）
第 60-100 局： 挑戰高難度成就（Ascension 15/20）
100+ 局：     補完剩餘成就 / 追求 Ascension 20 通關
```

## 6. 內容 Gating 機制（GLM 替代方案）

### 不使用「硬性數值牆」——使用三種內容 Gate

#### Gate A：配方 Gating（讓天賦樹成為「配方樹」）

在 talent 系統新增 5 個配方節點，取代純數值節點：
- `recipe_wood_fire`（cost 3）：解鎖焚林塔合成
- `recipe_fire_earth`（cost 3）：解鎖熔岩塔合成
- `recipe_earth_metal`（cost 3）：解鎖鍛造塔合成
- `recipe_yin_yang`（cost 5）：解鎖太極塔合成
- `recipe_mastery`（cost 3/級，3 級）：解鎖 Lv2 合成配方

**設計意圖**：玩家失敗後解鎖一個配方 → 下一局嘗試新配方 → 延長策略探索期

#### Gate B：機制學習（「見過 = 學會」）

- Wave 3 首次出現**護甲怪**（1 隻）+ UI 提示「此怪物有護甲，需暴擊塔」
- Wave 6 首次出現**再生怪**（1 隻）+ 提示
- Wave 10 首次出現**分裂怪**（1 隻）+ 提示
- Wave 15+ 組合詞條怪（護甲+再生）
- 對應天賦節點 `anti_armor`/`anti_regen`/`anti_split`：見過該詞條後可解鎖對抗策略

#### Gate C：五行失衡系統（中長期）

每局隨機地圖區域產生「能量場」：
- 「離火位」：火塔 +50% / 水塔 -50%
- 「坎水位」：水塔 +50% / 火塔 -50%
- 每 5 波切換一次
- Wave 10+ 出現「五行缺失怪」，僅在對應能量場內可造成傷害

### 7.2 結算天賦點流程

```ts
// 天賦點結算：確保每局都「有收穫」
function getEndGameTalentPoints(survivedWaves, personalBest, milestones, ascensionLevel): {
  // 基數（存活波次 / 4，保底 1）
  let points = max(1, floor(survivedWaves / 4));
  // PB 獎勵
  if (survivedWaves > personalBest) points += floor((survivedWaves - personalBest) / 2);
  // 首次里程碑獎勵（Wave 5/10/15/20）
  for (const m of [5, 10, 15, 20]) {
    if (survivedWaves >= m && !milestones.includes(m)) points += 3;
  }
  return points;
};
```
