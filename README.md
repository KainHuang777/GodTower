# 五行迷宮塔防 (Wuxing Maze Tower Defense)

> 以五行陰陽為核心的迷宮式塔防遊戲，玩家透過擺放砲台築起迷宮延長怪物路徑，並藉由跨局天賦系統持續強化能力。

---

## 🎮 遊戲簡介

**五行迷宮塔防**是一款執行於 Web 瀏覽器的純前端塔防遊戲，融合了：

- **迷宮建構**：砲台同時充當牆壁，標準關卡在 80×40 網格中建立迷宮，迫使怪物走最長路徑（A\* 動態尋路）
- **多地圖**：內建 4 張不同格局與難度的關卡地圖；教學與測試關卡另使用 20×10、64px 的全圖展示尺寸
- **五行陰陽系統**：7 種基礎屬性砲台（金木水火土 + 陰陽），含同系升級合成與異系配方合成（4 種活躍配方）
- **Meta 進度系統**：跨局天賦樹，支援多級升級，以 `localStorage` 持久化存儲
- **Roguelike 元素**：波間卡牌三選一（Common/Rare/Legendary）、起始隨機補給、神秘召喚、合成費用、詞條對策卡牌
- **Ascension 難度系統**：通關後解鎖分級難度（0-10 層），大幅延長遊戲壽命
- **像素美術**：塔與怪物以高密度像素精靈搭配程式化 fallback，維持外部資產失敗時仍可遊玩；主視覺採明亮日間山水與暖色道路

## 🕹️ 核心玩法

| 機制 | 說明 |
|:---|:---|
| **檢查點路徑** | 地圖設有 5 個必須按序訪問的檢查點，怪物需通過所有檢查點才能抵達基地 |
| **動態尋路** | 每次擺放或移除砲台後即時重算怪物路徑（A\*），確保路徑不被完全堵死 |
| **砲台合成** | 選取任意兩座砲台進行合成，同系升 Lv2，異系依五行相生關係產生特殊塔 |
| **路線引導** | 戰鬥中常駐繪製 25% 透明金色引導線與方向微型箭頭，依怪物行走路徑流動運作 |
| **五行相剋** | 金剋木 → 木剋土 → 土剋水 → 水剋火 → 火剋金；克制屬性傷害 **+30%** |

> **🎯 設計哲學**：本遊戲的核心樂趣是「**繞路迷宮建構 + 豐富合成路線探索**」。岩壁塔 3g 的低成本設計是有意為之——讓玩家能自由實驗迷宮佈局，透過逐步遭遇高難度關卡與怪物來學習繞路策略與合成搭配，而非在資源極度稀缺的壓力下被迫尋找唯一解。這是一款節奏從容的迷宮塔防，隨著 Ascension 層級提升逐步增加挑戰深度。

### 教學關卡目前流程

教學關卡固定顯示整張 20×10 試煉場，不需要拖曳鏡頭，依序引導：放置攻擊塔、放置連續岩壁、第一波戰鬥、塔合成、加速、波間隨機技能、空中敵人、Boss，最後進入天賦引導。複雜迷宮與長距離繞路保留給後續關卡。

### 地形素材狀態

目前已套用設計稿的明亮色盤、暖色道路、草地密度方向與檢查點光圈表現；設計稿中的草叢、石岸、水面仍是參考稿，尚未拆成正式可無縫平鋪的 runtime 素材。相關參考圖位於 `docs/mockups/terrain-material-board-v1.png`，後續會在不破壞尋路與效能的前提下逐步替換。

## 🎲 Roguelike 系統 (卡牌、詞條對策與召喚)

本遊戲內建豐富的 Roguelike 機制，為每局對戰帶來不同的局面策略與變化：
1. **波間卡牌三選一 (Rarity 稀有度系統)**：每波防守結束時，玩家可從隨機產生的 3 張卡牌中挑選 1 張增益。
   * **稀有度分級**：卡牌分為 **普通 (Common)**、**稀有 (Rare)** 與 **傳說 (Legendary)** 三種層級，並帶有專屬的呼吸發光動畫。
   * **波次權重**：隨波次推進，抽到高稀有度卡牌的機率會動態大幅提升（例如第 16 波後有 40% 的機率抽到傳說卡）。
   * **強效卡牌效果**：除屬性加成與資源外，更有「太極大共鳴（全體永久攻防提升）」、「全能工匠（下次建塔免費）」與「命運召喚（隨機獲得進階 Lv2 合成塔放置權）」等能扭轉戰局的強力卡牌。
2. **開局隨機補給 (Start Bonus)**：非教學/測試關卡下，開局隨機獲得 2 座不同元素的 Lv1 塔免費放置權，打破開局的單一建塔套路。
3. **神秘召喚 (Mystery Box)**：可花費金幣（價格依已解鎖塔的期望值動態計算）隨機召喚一座已解鎖的 Lv1 塔，具有極高的性價比。
4. **詞條對策卡牌 (Trait Counter)**：Wave 3/6/10 首次遭遇護甲/再生/分裂怪後，解鎖對應的詞條對策卡牌（每詞條 × 3 稀有度 = 共 9 張），包括「破甲一擊」、「灼燒標記」、「速攻指令」等可發動的戰鬥效果。
5. **波次獎勵金動態衰減**：為了避免後期金幣無限過剩，波次防禦成功獎勵套用動態衰減公式 `max(10, 15 + wave * 3 - floor(wave / 3) * 4)`，使 Wave 20 獎勵從原先的 75g 降低至 51g (-32%)。
6. **合成費用**：同系合成收取 `floor(min(a,b) × 0.5)`、異系合成收取 `floor((a+b) × 0.3)` 金幣。

## 🎖️ 天賦系統 (Meta-Progression)

- **天賦點獲取**：`天賦點 = max(1, floor(存活波次 / 4)) + 突破紀錄獎勵 + 里程碑首次獎勵`
- **多級升級**：所有天賦支援多次升級（最多 3～5 級），UI 顯示 `Lv.n/max`
- **雙軌天賦點制**：Track A（Ascension 0-5 刷關）僅基礎公式無額外獎勵；Track B（Ascension 6+ 挑戰）池含成就獎勵（單局最高 11 點）
- **開局金幣軟上限 (Soft Cap)**：天賦 `初始資金 I` 與 `初始資金 II` 提供的開局金幣上限設為 200g (原最高 310g)，保留開局初期的建設決策張力。
- **天賦感知 HP 難度補償因子**：怪物 HP 乘數在戰鬥中會套用已花費天賦點比例的額外加成（最高 +50% 額外 HP），使天賦點數極高的資深玩家仍可面對充足的遊戲威脅。
- **持久化**：天賦資料儲存於 `localStorage`，重開遊戲不會重置
- **路線分支**：基礎強化 / 攻擊強化 / 五行解鎖 / 陰陽解鎖 / 配方解鎖 / 詞條對策 六大路線

## 🏔️ Ascension 分級難度系統

通關 Wave 20 後解鎖 Ascension 難度層級（0-10，逐步擴展至 20）：
- 每層提高怪物 HP（`1.0 + A × 0.0875`）、速度（`1.0 + A × 0.0275`）、數量（`floor(A/2) × 0.05`）
- 高層級有額外詞綴（飛行怪提前、Boss 額外技能、金幣 -20%、負面事件等）
- 低 Ascension 層僅基礎天賦點公式；高 Ascension（6+）可獲得成就獎勵，實現雙軌天賦點制

## 🛠️ 技術規格

| 項目 | 內容 |
|:---|:---|
| **語言** | TypeScript |
| **打包工具** | Vite 5 |
| **測試框架** | Vitest（181 tests，6 files） |
| **渲染** | HTML5 Canvas |
| **精靈渲染** | OffscreenCanvas 預渲染快取，支援 16x16 與 16x24 2.5D 原生像素動畫與 45 度等角立體光影 |
| **視覺風格** | 明亮日間暖色山水 + 高密度像素精靈 + 深色描邊與柔和 Glow；夜色、星空、雷雨僅作特殊關卡主題 |
| **特效與後處理**| 高效 Outlines 邊框、受擊閃白快取、recoil 砲台後退、squash 怪物變形及拋物線/死亡 Splatters 特效 |
| **畫面適配** | 雙向等比例自適應縮放，搭載 CSS transform 縮放後的物理至邏輯座標精準映射對齊 |
| **音效系統** | 多通道 HTML5 Audio 靜態對象池（限制 10 實例避免洩漏）與背景音樂自動跳過錯誤 Fallback 機制 |
| **目標解析度** | 標準關卡 80×40 格（每格 16px = 1280×640）；教學/測試關卡 20×10 格（每格 64px = 1280×640） |
| **目標幀率** | 60 FPS |
| **同屏最大怪物** | 50+ |
| **同屏最大子彈** | 200+ |
| **外部依賴** | 無（零 runtime 依賴） |
